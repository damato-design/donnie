---
title: "Use Nested Modes Instead of New Tokens for Expressive Moments"
date: 2023-10-30
context: "Design systems struggle with expressive fragments, like highlighting a premium product on an otherwise uniform page. The instinct is to mint new tokens for each moment, but that bloats the token set and erodes consistency. Figma's 2023 introduction of Variables with Modes sharpened the question."
decision: "Scope expression with nested modes (Mise en Mode): visually box a region, then supply new values to existing semantic tokens only within that scope, rather than creating dedicated tokens per expressive moment."
alternatives:
  - option: "Create dedicated tokens for each expressive moment"
    pros:
      - "Explicit and easy to target"
    cons:
      - "Token sprawl"
      - "Erodes the bounded semantic vocabulary"
      - "Every new highlight is a new token to maintain"
  - option: "One-off component variants"
    pros:
      - "Localized to the component"
    cons:
      - "Doesn't compose across a region"
      - "Multiplies component surface area"
  - option: "Nested modes over existing semantic tokens"
    pros:
      - "Reuses the existing token vocabulary"
      - "Keeps the system consistent while allowing local expression"
      - "Scopes a 'mini-theme' to exactly where it's needed"
    cons:
      - "Requires tooling/mental model for scoping"
      - "Eventual-consistency of meaning within scopes"
reasoning: "Moving semantic responsibility from tokens to scopes resolves the tension between expression and consistency. A boxed region becomes a place for a new expression by receiving new values for the same semantic tokens, no new tokens required, so the system stays bounded and coherent."
whyItMattered: >-
  Mise en Mode reframed the question from "what token do I add?" to "what scope
  receives a new expression?" That shift is what lets a system stay consistent
  and still feel expressive where it counts.
