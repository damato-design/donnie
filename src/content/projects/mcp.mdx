Most AI tools answer questions about a person or a system from training data that is stale or simply made up. mcp.damato.design fixes that for my own corner of the web: it is a Model Context Protocol server that turns the damato.design ecosystem into resources an AI client can list and read directly.

Resources follow a `{subdomain}://{category}/{slug}` scheme, for example `donnie://projects/miseenmode` or `blog://posts/truly-semantic`, and are fetched from each site's `llms.txt` so the live sites remain the single source of truth. The server exposes three tools, `list_resources`, `get_resource`, and `get_llms_txt`, plus an `ask-damato` prompt for explicitly querying my content when several servers are connected.

It runs remotely, so connecting is a single URL with no install: point any MCP client (Claude Desktop, Cursor, Cline, Windsurf, Zed, Continue) at `https://mcp.damato.design/`. Source is on [GitHub](https://github.com/damato-design/mcp).
